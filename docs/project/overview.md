## LabOps 是什么

LabOps 是一个**轻量级开源运维平台**。它的核心能力是：

> **通过 Web 界面，实时监控和管理分布在不同网络环境中的多台设备。**

你可以把它理解为一个「私有的远程管理控制台」——在自己的服务器上部署中央服务，在需要管理的设备上安装轻量级 Agent，然后通过浏览器完成所有运维操作。

---

## 项目解决的问题

### 问题 1：多台设备管理分散

当你有多台服务器、虚拟机、开发机或树莓派需要管理时，传统做法是：

- 记住每台设备的 IP
- 逐个 SSH 登录
- 在每台设备上重复执行相同的命令
- 手动记录操作历史

LabOps 用一个 Web 界面统一管理所有设备，**一条命令可以同时下发到组内所有设备**。

### 问题 2：非技术人员难以操作

传统的运维方式需要 SSH 和命令行技能。LabOps 提供**图形化 Web 界面**：

- 点击鼠标就能执行预定义的命令模板
- 实时查看每台设备的 CPU、内存、磁盘使用率
- 所有操作自动记录审计日志，可追溯

### 问题 3：数据安全和隐私

使用第三方 SaaS 运维平台意味着你的设备数据、命令记录都存储在别人的服务器上。LabOps 支持**完全私有化部署**：

- 中央服务部署在你自己的服务器上
- 所有数据（设备信息、任务记录、审计日志）存储在你的服务器本地
- 你拥有数据的完全控制权
- 不需要将任何数据发送给第三方

### 问题 4：不同网络环境的设备难以统一接入

你的设备可能分布在：

- 云服务器（有公网 IP）
- 家庭局域网（NAT 后面）
- 公司内网（有防火墙）
- 不同的网络供应商

LabOps 的 Agent 通过 WebSocket **主动连接**中央服务（出站连接），不需要中央服务能访问 Agent。只要 Agent 所在设备能访问互联网，就能被管理。

## 核心功能

### Agent-Server-Web 实时控制闭环

```
你的浏览器               中央服务                被管理设备
(Web 控制台)    ←→    (Go 服务端)    ←→    (Agent 进程)
 查看状态              处理逻辑              上报信息
 下发命令              存储数据              执行命令
```

**这个闭环中的每一步都是真实的：**

- Agent 每 10 秒向服务端发送心跳，包含实时 CPU/内存/磁盘使用率
- 你在 Web 界面创建命令 → 服务端通过 WebSocket 推送给 Agent → Agent 执行 → 返回 stdout/stderr/退出码/耗时
- 所有操作（连接、注册、命令下发、执行结果）记录完整的审计日志

### 设备管理

| 功能 | 说明 |
|------|------|
| 设备清单 | 可搜索的设备列表，展示 OS、IP、主机名、CPU/内存/磁盘规格 |
| 实时指标 | CPU 使用率、内存使用率、磁盘使用率，自动刷新 |
| 心跳追踪 | 35 秒超时自动标记离线 |
| 分组管理 | 将设备组织到逻辑分组，显示在线率 |

### 任务系统

| 功能 | 说明 |
|------|------|
| 临时命令 | 在任意设备上执行任意命令（需要确认键防止误操作） |
| 命令模板 | 预定义的参数化命令，支持枚举/正则/范围参数校验 |
| 批量下发 | 一键向组内所有在线设备发送命令 |
| 结果查看 | 展开查看 stdout/stderr/退出码/执行耗时 |

### AI 运维

| 功能 | 说明 |
|------|------|
| 健康评分 | 基于规则的 0-100 分（CPU/内存/磁盘阈值、离线检测） |
| LLM 分析 | 可选接入 OpenAI 兼容或 Anthropic API 进行智能分析（需配置 API Key） |
| 推荐命令 | LLM 根据设备状态推荐运维命令 |
| 自动执行 | 可开启自动执行只读命令（修改系统的命令需人工确认） |

### 安全机制

| 机制 | 实现 |
|------|------|
| 用户认证 | Session Cookie（HttpOnly + Secure + SameSite=Strict） |
| CSRF 保护 | 双重提交 Cookie 模式 |
| 密码安全 | bcrypt（cost 12），最少 12 字符，首次登录强制改密 |
| 设备认证 | 一次性接入码 → 每设备独立 256 位密钥 |
| 权限控制 | 三级角色（管理员/操作员/观察者），细粒度权限 |
| 密钥加密 | AES-256-GCM 加密存储 LLM API Key |
| 速率限制 | 按 IP 令牌桶（登录 5 次/3 分钟，接入 10 次/分钟） |
| 危险命令拦截 | 自动过滤 rm -rf、mkfs、dd、shutdown 等 |

## 不做什么（非目标）

LabOps 有明确的边界：

- ❌ 不是大规模监控平台（不替代 Zabbix/Prometheus）
- ❌ 不是远程桌面工具（不替代 TeamViewer/VNC）
- ❌ 不是配置管理工具（不替代 Ansible/Puppet）
- ❌ 不是容器编排平台（不替代 Kubernetes）
- ❌ 不支持集群/高可用部署（当前是单实例架构）
- ❌ 不支持多租户（一个实例管理一套设备）

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 18 + TypeScript 5.6 + Vite 5.4 | 12 个功能页面 |
| UI 组件 | Ant Design 5.21 | 中文界面 |
| 状态管理 | Zustand 4.5 | 极简状态管理 |
| 后端 | Go 1.25 + net/http | 仅使用标准库，无外部框架 |
| WebSocket | gorilla/websocket v1.5.3 | Agent 通信 |
| 数据库 | MySQL 8.0（生产）/ SQLite（开发） | 方言抽象，双后端支持 |
| Agent | Go 1.24 + gopsutil v4 | 单文件静态编译 |
| 反向代理 | Nginx 1.27 | TLS 终止 + 静态文件服务 |
| 容器化 | Docker + Docker Compose | 三服务编排 |
| CI | GitHub Actions | Go vet/test + TypeScript check/build |
