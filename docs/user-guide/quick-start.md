# 快速开始

> **目标读者:** 首次使用 LabOps 的用户
> **前置阅读:** 项目已部署并可通过浏览器访问

---

## 1. 登录系统

1. 浏览器打开 `https://你的域名`（或 `http://你的IP`）
2. 输入用户名 `admin`
3. 输入部署时设置的 `LABOPS_BOOTSTRAP_ADMIN_PASSWORD`

【截图：登录页面】

### 首次登录强制改密

第一次登录时，系统会强制要求修改密码。新密码要求：

- 至少 12 个字符
- 不能与初始密码相同

> **提示：** 改密后所有其他设备的会话会失效，这是安全设计。

## 3. 添加第一台设备

### 步骤 1：生成接入码

1. 点击左侧「**设备接入**」
2. 点击「**生成接入码**」
3. 设置有效期（建议 30 分钟）和最大使用次数（建议 1 次）
4. 点击确定，**立即复制接入码**

【截图：生成接入码对话框】

> ⚠️ 接入码只会显示一次，请立即复制保存。

### 步骤 2：在目标设备上安装 Agent

SSH 登录到你要管理的设备，执行：

```bash
# 编译 Agent（或在开发机上编译后上传）
cd /path/to/LabOps/agent
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o labops-agent ./cmd/agent

# 使用安装脚本
sudo bash scripts/install-agent.sh \
  --server "https://你的域名" \
  --enroll-code "复制的接入码" \
  --name "$(hostname)" \
  --group "我的设备"
```

详细说明见 [Agent 部署详解](../deployment/agent-deployment.md)。

### 步骤 3：确认设备上线

1. 回到 Web 界面，点击「**设备管理**」
2. 应该能看到新设备，状态显示为 **在线** ✅
3. 实时 CPU、内存、磁盘数据每 10 秒自动更新

【截图：设备列表页面，显示一台在线设备】

## 4. 执行第一条命令

### 查看设备信息

1. 点击设备名称进入详情页
2. 在「临时命令」输入框中输入：
   ```
   hostname
   ```
3. 输入确认键 `EXECUTE`（防止误操作）
4. 点击执行

**预期结果：** 任务表格中出现新任务，展开可以看到命令输出。

【截图：设备详情页，显示命令执行结果】

### 使用命令模板

1. 进入「**任务管理**」
2. 选择「模板命令」标签
3. 选择一个模板（如「主机名」「系统运行时间」）
4. 选择目标设备或分组
5. 点击执行

## 5. 数据刷新频率

| 页面 | 自动刷新间隔 | 说明 |
|------|:----------:|------|
| 仪表盘 | 10 秒 | 设备统计、最近任务 |
| 设备列表 | 10 秒 | 设备状态和指标 |
| 设备详情 | 3 秒 | 实时 CPU/内存/磁盘进度条 |
| 任务列表 | 3 秒 | 任务状态更新 |
| 审计日志 | 15 秒 | 操作记录 |
| AI Ops | 30 秒 | 健康分析报告 |
