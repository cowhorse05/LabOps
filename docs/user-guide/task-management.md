# 任务管理

> **目标读者:** 日常运维用户
> **前置阅读:** [设备管理](device-management.md)

---

## 创建任务

导航到「**任务管理**」，可以创建命令任务。

### 临时命令

直接在目标设备上执行 Shell 命令。

**操作步骤：**

1. 选择「临时命令」
2. 输入要执行的命令（如 `df -h`、`free -m`、`systemctl status nginx`）
3. 选择目标（单台设备或按分组批量下发）
4. 在确认框中输入 `EXECUTE`
5. 点击「执行」

**权限要求：** `commands:adhoc`

### 命令模板

使用预定义的参数化模板执行命令。模板由管理员在「命令模板」页面管理。

**操作步骤：**

1. 选择「模板命令」
2. 从下拉菜单选择一个模板（如「磁盘使用情况」「系统运行时间」）
3. 如有参数，填写参数值（系统会自动校验参数格式）
4. 选择目标（单台设备或按分组）
5. 点击「执行」

**权限要求：** `templates:execute`

**内置模板（Linux）：**

| 模板名称 | 命令 | 说明 |
|---------|------|------|
| 主机名 | `hostname` | 查看设备主机名 |
| 系统运行时间 | `uptime` | 查看系统运行时长和负载 |
| 磁盘使用情况 | `df -h` | 查看磁盘空间 |
| 内存使用情况 | `free -h` | 查看内存使用 |
| 进程摘要 | `ps aux --sort=-%mem \| head -20` | 查看内存占用前 20 的进程 |

## 批量下发

当选择「按分组」时，任务会下发给该组内**所有在线设备**。每台设备独立执行，各自返回结果。

**适用场景：**
- 「所有 Web 服务器重启 Nginx」
- 「所有数据库服务器检查磁盘空间」
- 「所有开发机更新 hosts 文件」

## 查看任务结果

### 任务列表

任务列表展示所有任务，包括：

- 任务 ID
- 目标设备
- 命令内容
- 执行状态
- 请求者
- 创建时间

任务列表每 3 秒自动刷新。

### 任务状态

| 状态 | 含义 |
|------|------|
| pending | 已创建，等待 Agent 上线后下发 |
| running | 已下发，Agent 正在执行 |
| success | 执行完成，退出码为 0 |
| failed | 执行完成，退出码不为 0 |
| timeout | 执行超时（超过 timeout_seconds） |

### 查看输出

点击任务行的展开按钮，可以查看：
- **stdout** — 命令的标准输出（最多 256KB）
- **stderr** — 命令的标准错误输出（最多 256KB）
- **退出码** — 命令的退出码（0 = 成功）
- **执行耗时** — 从下发到返回结果的毫秒数

## 任务安全机制

| 机制 | 说明 |
|------|------|
| 确认键 | 临时命令必须输入 `EXECUTE` 才执行 |
| 权限控制 | ad_hoc 需 `commands:adhoc`，template 需 `templates:execute` |
| 超时限制 | 命令最长执行 300 秒，超时自动终止 |
| 输出截断 | stdout/stderr 各限制 256KB |
| 审计记录 | 所有任务创建和执行结果自动记录审计日志 |

---

## 审计日志

导航到「**审计日志**」查看所有操作记录。

每 15 秒自动刷新。记录内容包括：

- 操作者（用户名）
- 操作类型（登录、命令创建、命令完成、设备接入等）
- 目标设备
- 关联任务
- 操作结果
- 操作时间
- 来源 IP
