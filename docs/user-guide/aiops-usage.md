## AI Ops 报告

导航到「**AI 运维**」页面，可以看到设备的健康分析报告。

【截图：AI Ops 页面】

### 报告包含什么

| 部分 | 说明 |
|------|------|
| 健康评分 | 每台设备的 0-100 分健康评分 |
| 扣分原因 | 哪些指标触发了扣分 |
| 设备洞察 | 每个设备的详细状态卡片 |
| 分组概览 | 各分组的设备数和在线率 |

### 健康评分规则

每台设备初始 100 分，根据以下规则扣分：

| 条件 | 扣分 |
|------|:---:|
| 设备离线 | -40 |
| CPU 使用率 > 80% | -20 |
| CPU 使用率 > 60% | -10 |
| 内存使用率 > 80% | -20 |
| 内存使用率 > 60% | -10 |
| 磁盘使用率 > 85% | -15 |
| 任务失败率 > 50% | -20 |
| 任务失败率 > 20% | -10 |

## LLM 智能分析

### 配置 LLM

1. 导航到「**AI 运维设置**」
2. 填写 LLM 配置：

| 配置项 | 说明 | 示例 |
|-------|------|------|
| 提供商 URL | LLM API 地址 | `https://api.openai.com` 或 `https://api.anthropic.com` |
| API Key | LLM API 密钥 | `sk-...` 或 `sk-ant-...` |
| 模型 | 模型名称 | `gpt-4o`、`claude-sonnet-4-6` 等 |
| 提供商类型 | OpenAI 兼容 或 Anthropic | 选择对应类型 |

3. 点击「**测试连接**」验证配置正确
4. 开启「**启用 AI 分析**」

【截图：AI Ops 设置页面】

### 支持的 LLM

| 提供商 | 类型 | API 格式 |
|--------|------|---------|
| OpenAI | `openai` | `/v1/chat/completions` |
| Anthropic | `anthropic` | `/v1/messages` |
| 任何 OpenAI 兼容 API | `openai` | `/v1/chat/completions` |

> **安全说明：** LLM API Key 使用 AES-256-GCM 加密存储在数据库中。加密密钥是部署时设置的 `LABOPS_ENCRYPTION_KEY`。

## LLM 推荐命令

AI 运维分析完成后，LLM 会根据设备状态推荐运维命令。

### 查看推荐

在 AI Ops 页面，推荐以卡片形式展示，包含：

- 目标设备
- 推荐命令
- 推荐原因
- 优先级（高/中/低）
- 是否会修改系统状态（isMutation）

### 执行推荐

点击推荐卡片上的「**执行**」按钮，命令会作为任务下发到目标设备。

### 自动执行模式

开启「**自动执行只读命令**」后，LLM 推荐的安全命令（isMutation = false）会自动下发执行，无需人工确认。

会修改系统状态的命令（isMutation = true）始终需要人工确认。
