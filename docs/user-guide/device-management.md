# 设备管理

> **目标读者:** 日常运维用户
> **前置阅读:** [快速开始](quick-start.md)

---

## 设备列表页

导航到「**设备管理**」，可以看到所有已接入的设备。

【截图：设备列表页面】

### 设备状态

| 状态 | 含义 |
|------|------|
| 🟢 在线 | Agent 正常连接，心跳在 35 秒内 |
| 🔴 离线 | Agent 断开或心跳超时（35 秒无心跳） |

### 搜索和筛选

在搜索框中输入设备名称或 IP 可以快速定位设备。

### 设备信息

列表展示每台设备的：
- 设备名称和 IP
- 操作系统
- 分组
- CPU/内存/磁盘使用率
- 最后心跳时间
- 凭据状态

### 创建设备

点击「**创建设备**」可以手动注册设备（通常由 Agent 自动注册）。

### 删除设备

点击删除按钮可移除设备记录。

### 吊销凭据

点击「**吊销**」可立即使设备的认证凭据失效。吊销后，Agent 将无法连接服务端，直到重新接入。

---

## 设备详情页

点击设备名称进入详情页。

【截图：设备详情页面】

### 资产信息

显示设备的完整信息：
- Agent ID、名称、分组
- 操作系统和版本
- 主机名
- IP 地址
- CPU 核心数、总内存、总磁盘空间

### 实时指标

三个进度条实时展示：
- **CPU 使用率** — 当前 CPU 占用百分比
- **内存使用率** — 当前内存占用百分比
- **磁盘使用率** — 当前磁盘占用百分比

数据每 3 秒自动刷新。进度条颜色：
- 🟢 0-60%：正常
- 🟡 60-80%：注意
- 🔴 80-100%：告警

### 临时命令执行

在下方的文本框中输入命令，可以在这台设备上执行任意 Shell 命令。

**安全约束：**
- 必须在确认框中输入 `EXECUTE` 才能执行
- 该功能需要 `commands:adhoc` 权限（管理员和操作员可用，观察者不可用）

### 任务历史

该设备的历史任务列表。可展开查看每条任务的 stdout 和 stderr 输出。

---

## 分组管理

导航到「**分组管理**」，可以按分组查看设备。

【截图：分组管理页面】

每个分组显示：
- 分组名称
- 设备总数
- 在线设备数
- 在线率

分组由 Agent 安装时的 `--group` 参数决定。同一台服务器的 Agent 可以归入同一分组，方便批量操作。

---

## 常用设备管理操作

| 操作 | 路径 | 说明 |
|------|------|------|
| 查看所有设备 | 设备管理 | 设备列表 + 实时状态 |
| 查看设备详情 | 点击设备名 | 完整信息 + 实时指标 |
| 执行命令 | 设备详情 → 临时命令 | 需要 `commands:adhoc` 权限 |
| 查看任务历史 | 设备详情 → 任务表格 | 可展开看 stdout/stderr |
| 创建设备 | 设备管理 → 创建设备 | 手动注册 |
| 删除设备 | 设备管理 → 删除 | 移除设备记录 |
| 吊销凭据 | 设备管理 → 吊销 | 使 Agent 无法连接 |
| 按分组查看 | 分组管理 | 分组统计 |
